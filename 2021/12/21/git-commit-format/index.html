git commit message 规范学习

发表于 2021-12-21 更新于 2021-12-22 分类于 配置环境

在编写开源项目(把github当云盘)时候，每次都很纠结git commit的信息怎么写，所以特意学习一下如何编写git commit log

书写规范

查阅多方资料后，有一个被大家所广泛接受的规范是Angular团队的git提交规范1, 大致框架如下，其中第一行被合称为header

Revert

首先是一个特殊情况，如果该提交是某次对之前版本的回滚，则应该在第一行前加上revert:，紧接着是回溯版本的header（即第一行），随后在body中加上This reverts commit <hash>，hash中加入回滚版本的SHA编码。

Type

type应该从以下几种中选择（这里复制的源文档，应该都能看懂）

Scope

范围是可选的，是用来制定更改位置的内容，例如 $location、$browser、$compile、$rootScope、ngHref、ngClick、ngView 等。如果有多个的话可以用*来代替

Subject

主题中是对commit简洁的描述

要遵循一下规则

  • 要是用祈使句和一般现在时
  • 第一个字母不要大写
  • 结尾不要有句号

Body

规则与Subject中相同，但是内容应更详细，包括更改的动机，以及与更改前进行比较

footer

页脚应该包括任何和重大修改有关的信息，包括与这条commit相关的issue。

插件

vscode中有大佬们开发的自动生成满足上述规范的插件git-commit-plugin2，使用起来很简单，各位可以自己去看文档。
